# Master Roadmap

DataCenterDigitalTwin is intended to grow from a local telemetry simulator into a realistic, cloud-native AI data center digital twin.

## Phases

- [Phase 1: Production-Quality Repository](phase-1-production.md)
- [Phase 2: Grid-Aware Data Center Simulation](phase-2-grid-aware-simulation.md)
- [Phase 3: Kubernetes Learning and Deployment](phase-3-kubernetes.md)
- [Phase 4: Cloud-Native Digital Twin](phase-4-cloud-native.md)
- [Phase 5: Home Lab Deployment](phase-5-home-lab.md)

## Phase Summary

### Phase 1 — Production-Quality Repository
Prepare the repository with clean structure, tests, linting, CI, documentation, and a professional engineering workflow.

### Phase 2 — Grid-Aware Data Center Simulation
Expand the simulator into a realistic AI data center model with utility pricing, electrical constraints, GPU loads, liquid cooling, PUE, alerting, ClickHouse telemetry, Grafana, React, and FastAPI controls.

### Phase 3 — Kubernetes Learning and Deployment
Use the project as the application for learning Kubernetes, Helm, kind, k3d, CI-based deployment testing, and observability.

### Phase 4 — Cloud-Native Digital Twin
Convert the platform into a Kubernetes-native system with scalable services, ingestion workers, rolling deployments, health checks, alerting, and production-style operations.

### Phase 5 — Home Lab Deployment
Run the system on a cheap always-on K3s home lab using a mini PC, persistent storage, Grafana, MQTT, backups, and remote access.

## Long-Term Goal

Build a portfolio-grade platform demonstrating controls engineering, electrical and cooling simulation, data engineering, Kubernetes, CI/CD, observability, distributed systems, and AI-assisted data center operations.
